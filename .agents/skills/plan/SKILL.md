# Planning Protocol

When asked to create a plan for this repository, you **MUST** adhere to the following strict guidelines:

1. **Use the Template**: All plans must be formatted according to the official repository template. You can find this template at `docs/plan_template.md`. Read it first to ensure exact compliance.
2. **File Location**: Save the newly created plan as a Markdown file inside the repository's `docs/` directory (e.g., `docs/plan_my_feature.md`).
3. **NO BRAIN ARTIFACTS**: **CRITICAL:** Do **NOT** create artifacts in the agent's brain directory. All plans and walkthroughs must live strictly as Markdown files in the project repository.
4. **Follow Repo Standards**: When devising your tasks and milestones, ensure the plan adheres to the codebase's clean code principles (DRY, KISS, YAGNI, composition over inheritance, loose coupling, separation of concerns).
